# Git 브랜치 전략 비교

Git Flow는 master, develop, feature, release, hotfix 브랜치를 사용하는 체계적인 모델이다. 대규모 프로젝트에 적합하지만 복잡성이 높다. GitHub Flow는 main과 feature 브랜치만 사용하며 PR 기반으로 단순하다. Trunk Based Development는 모든 개발자가 main에 직접 커밋하며, 피처 플래그로 기능을 제어한다. CI/CD가 잘 갖춰진 팀에 적합하다.
