# CI/CD 파이프라인 설계

CI(지속적 통합)는 코드 변경을 자주 통합하고 자동 빌드·테스트하는 프랙티스다. CD(지속적 배포)는 테스트를 통과한 코드를 자동으로 프로덕션에 배포한다. GitHub Actions, Jenkins, GitLab CI 등의 도구를 사용한다. 파이프라인에는 린트, 유닛 테스트, 빌드, 스테이징 배포, E2E 테스트, 프로덕션 배포 단계가 포함된다. 블루-그린 배포와 카나리 배포로 위험을 최소화한다.
