# 테스트 피라미드 전략

테스트 피라미드는 단위 테스트를 가장 많이, 통합 테스트를 중간, E2E 테스트를 가장 적게 작성하라는 전략이다. 단위 테스트는 빠르고 격리되어 있어 피드백 루프가 짧다. 통합 테스트는 모듈 간 상호작용을 검증한다. E2E 테스트는 실제 사용자 시나리오를 시뮬레이션하지만 느리고 불안정할 수 있다. TDD는 테스트를 먼저 작성하고 코드를 구현하는 방법론이다.
