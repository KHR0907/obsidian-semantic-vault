# 디자인 패턴 실전 활용

싱글톤은 클래스의 인스턴스를 하나만 유지하며, 데이터베이스 연결 풀에 자주 사용된다. 옵저버 패턴은 상태 변화를 구독자에게 알리며 이벤트 시스템의 기반이다. 팩토리 패턴은 객체 생성 로직을 캡슐화하고, 전략 패턴은 알고리즘을 런타임에 교체할 수 있게 한다. 어댑터 패턴은 호환되지 않는 인터페이스를 연결한다.
