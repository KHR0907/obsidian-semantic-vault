# SSR vs CSR vs SSG 렌더링 전략

서버 사이드 렌더링(SSR)은 서버에서 HTML을 생성하여 초기 로딩이 빠르고 SEO에 유리하다. 클라이언트 사이드 렌더링(CSR)은 브라우저에서 자바스크립트로 UI를 구성하며 SPA에 적합하다. 정적 사이트 생성(SSG)은 빌드 시점에 HTML을 미리 생성하여 CDN으로 배포한다. Next.js는 세 가지를 페이지별로 혼합할 수 있는 하이브리드 접근을 제공하고, ISR로 정적 페이지의 재생성도 지원한다.
