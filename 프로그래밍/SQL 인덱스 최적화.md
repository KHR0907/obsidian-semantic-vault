# SQL 인덱스 최적화

인덱스는 데이터베이스 검색 속도를 높이는 자료구조로, B-트리가 가장 일반적이다. WHERE, JOIN, ORDER BY에 사용되는 컬럼에 인덱스를 생성하면 효과적이다. 복합 인덱스는 컬럼 순서가 중요하며 최좌측 접두사 규칙을 따른다. 커버링 인덱스는 필요한 모든 컬럼이 인덱스에 포함되어 테이블 접근 없이 쿼리를 처리한다. 과도한 인덱스는 쓰기 성능을 저하시킨다.
