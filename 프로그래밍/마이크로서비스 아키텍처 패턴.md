# 마이크로서비스 아키텍처 패턴

마이크로서비스는 각 서비스가 독립적으로 배포, 확장, 개발되는 아키텍처다. API 게이트웨이는 클라이언트 요청을 적절한 서비스로 라우팅한다. 서비스 간 통신은 REST, gRPC, 메시지 큐로 이루어진다. 사가 패턴은 분산 트랜잭션을 관리하고, 서킷 브레이커는 장애 전파를 차단한다. 서비스 메시를 사용하면 네트워크 레벨의 관리가 투명해진다.
