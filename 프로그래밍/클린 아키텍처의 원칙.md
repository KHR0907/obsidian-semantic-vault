# 클린 아키텍처의 원칙

클린 아키텍처는 의존성이 바깥에서 안쪽으로만 향하는 동심원 구조다. 가장 안쪽의 엔티티는 비즈니스 규칙을, 유스케이스는 애플리케이션 로직을 담는다. 인터페이스 어댑터 계층에서 DB, UI, 외부 서비스와의 변환을 처리한다. 이 구조를 따르면 프레임워크나 DB 변경이 비즈니스 로직에 영향을 미치지 않는다.
