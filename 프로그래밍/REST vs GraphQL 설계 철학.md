# REST vs GraphQL 설계 철학

REST는 리소스 중심 설계로 각 엔드포인트가 특정 리소스를 나타낸다. 표준 HTTP 메서드(GET, POST, PUT, DELETE)로 CRUD를 수행한다. GraphQL은 클라이언트가 필요한 데이터만 정확히 요청할 수 있는 쿼리 언어다. 오버페칭과 언더페칭 문제를 해결하지만, 캐싱이 더 복잡하고 서버 구현 비용이 높다.
